17.2 Grundlagen

17.2.1 ChatGPT ist auch ein Transformer

👨‍🏫 Bist du ein Transformer?

🤖 Ja, ich bin ein Modell, das auf der GPT-3.5-Architektur von OpenAI basiert, und diese Architektur wird als “Transformer” bezeichnet. Der Begriff “Transformer” bezieht sich auf die spezifische Architektur des neuronalen Netzwerks, die von Vaswani et al. in ihrem Paper “Attention is All You Need” vorgestellt wurde. Die Transformer-Architektur hat sich als äußerst leistungsfähig für natürliche Sprachverarbeitungsaufgaben erwiesen und bildet die Grundlage für Modelle wie GPT (Generative Pre-trained Transformer), einschließlich GPT-3.5, auf dem meine Fähigkeiten basieren.

[Quelle: ChatGPT-3.5, 2023-11-23]

17.2.2 Alles ist API

Die Python- (oder R-)Pakete zum Zugriff zur ChatGPT-API sind letztlich nur Wrapper um den “direkten” Zugriff auf die API.

Definition 17.1 (API (Application Programming Interface)) Eine API stellt genormte Befehle zur Verfügung, um von einem Dienst (über eine Webseite) Daten abzufragen oder einzustellen. \(\square\)

17.2.3 curl

curl ist eine gängige Methode (auf der Kommondozeile), um Daten von einer Webseite abzufragen (oder einzustellen).

Laut Dokumentation der API lautet der CURL-Befehl wie folgt (Pseudocode):

```{zsh}
#| eval: false
curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -d '{
  "model": "gpt-3.5-turbo",
  "messages": [{"role": "user", "content": "Was ist die Frage zur Antwort '42'?"}] ,
  "temperature": 0,
  "max_tokens": 256
}'
```

Entscheidend ist der “Endpunkt” der URL: completions.

Hinweis

OpenAi stellt eine Reihe von spezialisierten Diensten zur Verfügung, z.B. zur Sentimentanalyse von Tweets oder, natürlich, Textgeneration, und vieles mehr. \(\square\)

Obige Syntax übersetzt sich so nach Python:

# This code is for v1 of the openai package: pypi.org/project/openai
from openai import OpenAI
client = OpenAI()

response = client.chat.completions.create(
  model="gpt-3.5-turbo",
  messages=[],
  temperature=0,
  max_tokens=256
)

17.2.4 Prompting

Als Prompt kann man z.B. übergeben (bezeichnet als “System”):

🧑‍🦰 You will be provided with a tweet, and your task is to classify its sentiment as positive, neutral, or negative. USER

Dann kommt der zu klassifizierende Textschnipsel (bezeichent als “user”):

📄 I loved the new Batman movie!

Und schließlich antwortet der Bot:

🤖 positive

Es ist günstig, dem Bot zu sagen, in welcher Sprache der Tweet ist. Außerdem ist es nützlich, den Prompt (die Anweisung) bereits in der Zielsprache zu formulieren.

prompt_stem <- "Nach dem Doppelpunkt folgt ein Tweet in deutscher Sprache. Klassifizieren Sie das Sentiment dieses Tweets als positiv, neutral oder negativ. Antworten Sie nur mit einem Wort. Hier ist der Tweet: "
prompts <- 
  str_c(prompt_stem, tweets)

17.2.5 Anmelden an der API

Die API erlaubt nur Zugriffe angemeldeter Nutzer.

OPENAI_API_KEY <- Sys.getenv("OPENAI_API_KEY")

Damit eine Environment-Variable OPENAI_API_KEY ausgelesen werden kann, muss sie in .Rprofile definiert sein. Alternativ kann man aber die Variable auch auf anderen Wegen definieren, etwa aus einer Textdatei einlesen.

Wichtig

Lassen Sie sensible Daten, wie API-Keys, niemals auf öffentlichen Ordnern oder Repos (etwa auf Github) herumliegen. Stellen Sie sich vor, Sie haben bei dem Dienst ihre Kreditkarte hinterlege und ein übelwollender Dritter nutzt kostenpflichtige Dienste mit sehr hohem Budget. 🤯 \(\square\)

17.3.1 Curl-Wrapper in eine Funktion gebracht

ask_chatgpt <- function(prompt) {
response <- POST(
  # curl https://api.openai.com/v1/chat/completions 
  url = "https://api.openai.com/v1/chat/completions", 
  # -H "Authorization: Bearer $OPENAI_API_KEY"
  add_headers(Authorization = 
                paste("Bearer", Sys.getenv("OPENAI_API_KEY"))),
  # -H "Content-Type: application/json"
  content_type_json(),
  # -d '{
  #   "model": "gpt-3.5-turbo",
  #   "messages": [{"role": "user", "content": "What is a banana?"}] 
  # }'
  encode = "json",
  body = list(
    model = "gpt-3.5-turbo",
    messages = list(list(role = "user", content = prompt))
  ))
  str_trim(content(response)$choices[[1]]$message$content)
}
