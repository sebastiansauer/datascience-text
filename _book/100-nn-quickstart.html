14.2 Quick-Start

14.2.1 MNIST: Hello, Deep Learning

Der MNIST-Datensatz ist eine Sammlung handgeschriebener Ziffern (bzw. deren digitale Repräsentation in Matrixform), s. Abbildung fig-mnist

Das maschinelle Erkennen von handgeschriebenen Ziffern ist nicht so einfach; für Menschen ist es eine mühelose Angelegenheit, dank der erstaunlichen Fähigkeiten unseres visuellen Cortex (und dem Evolutionsvorteil des visuellen Erkennens, der unseren visuellen Corext “trainiert” hat).

Das Erkennen der Ziffern mit Hilfe von Deep Learninng ist eine Einstiegsaufgabe, sozusagen das “Hello, World” des Deep Learnings.

Der MNIST-Datensatz umfasst 70k Bilder und haben eine Auflösung von 28*28=784 Pixeln in 8 Bit Graustufen (also \(2^8 = 256\) Werte).

14.2.2 Quick Start mit R

Wir halten uns an das Tutorial von TensforFlow for R, “Hello, World!”.

14.2.2.1 Setup

Wir starten die benötigten Pakete:

library(keras)  # TensorFlow API
library(tensorflow)  # TensorFlow pur
library(tidyverse)  # Datenjudo
library(tictoc)  # Zeitmessung

Das Installieren von TensorFlow bzw. Keras kann Schwierigkeiten bereiten. Tipp: Stellen Sie in RStudio sicher, dass Sie die richtige Python-Version verwenden.

mnist <- dataset_mnist()
X_train <- mnist$train$x
X_test <- mnist$test$x
y_train <- mnist$train$y
y_test <- mnist$test$y

In Kurzform kann man synonym schreiben:

c(c(x_train, y_train), c(x_test, y_test)) %<-% keras::dataset_mnist()

14.2.2.3 Neuronales Netz 1

Für unser Netzwerk wollen wir Werte zwischen 0 und 1, daher teilen wir durch den Max-Wert, d.i. 255:

x_train <- X_train / 255
x_test <-  X_test / 255
model <- keras_model_sequential(input_shape = c(28, 28)) %>%
  layer_flatten() %>%
  layer_dense(128, activation = "relu") %>%
  layer_dropout(0.2) %>%
  layer_dense(10)

14.2.2.4 Best-Bet-Digit

Möchte man ein Modell, das gleich die “Best-Bet-Digit” nennt, kann man das so machen:

predictions <- model %>%
  predict(X_test[1:5, , ]) %>%  # nur die ersten 5
  k_argmax()
## 1/1 - 0s - 65ms/epoch - 65ms/step

predictions$numpy()
## [1] 7 2 1 0 4

Oder ein eigenes, dazu passendes Modell bauen:

probability_model <- 
  keras_model_sequential() %>%
  model() %>%
  layer_activation_softmax() %>%
  layer_lambda(tf$argmax)

Hier sind die Vorhersagen:

probability_model(x_test[1:5, , ])
## tf.Tensor([3 2 1 0 4 2 2 0 2 4], shape=(10), dtype=int64)
y_test[1:5]
## [1] 7 2 1 0 4
